# Python Data Profiler

A lightweight desktop GUI application for quickly profiling CSV and Parquet datasets. Select a file, click Analyse, and get statistical summaries, plain-English findings, and export options — all without writing any code.

---

## Features

- **Tkinter GUI** — file browser, results panel, export buttons; no command line needed
- Loads **CSV** and **Parquet** files into a pandas DataFrame
- Computes per-column statistics including:
  - Null count and percentage
  - Unique count and percentage
  - Min, max, mean, median, mode, standard deviation
  - Q1 (25th percentile) and Q3 (75th percentile)
  - Skewness, zero count, negative count *(numeric columns)*
  - Top 5 most frequent values, average string length *(categorical/boolean columns)*
- Dataset-level summary: total rows, columns, nulls, duplicate rows, memory usage
- **Findings & Recommendations** — plain-English interpretations generated by a rule-based engine:
  - Primary key and composite key candidates
  - Per-column data quality warnings and usage notes
- **Export** results as JSON or CSV with a single click
- **Analysis timing** — elapsed time shown at the top of every result
- Rich-formatted output (Unicode table borders, aligned columns) in the results panel

## Usage

Launch the application:

```bash
python main.py
```

A window will open with the following controls:

1. **Browse** — opens a file dialog filtered to `.csv` and `.parquet` files; the selected path is shown in the entry field
2. **Analyse** — loads the file, computes statistics, runs the findings engine, and displays results in the panel below; elapsed time is shown at the top
3. **Export JSON** / **Export CSV** — save the results to a file (enabled only after a successful analysis)

The results panel is scrollable both vertically and horizontally, and re-renders at the current window width when Analyse is clicked.

## Findings & Recommendations

The findings section appears above the statistics tables and is generated by a rule-based engine in `interpreter.py`. No external AI service is used — all analysis is local.

### Table-level findings

| Rule | Example output |
|---|---|
| Column with 100% unique non-null values | `PRIMARY KEY CANDIDATE: 'customer_id' — 100% unique, no nulls` |
| No single-column PK; pair/triple of columns is jointly unique | `COMPOSITE KEY CANDIDATE: ('col_a', 'col_b') — unique together` |
| Duplicate rows present | `Duplicate rows detected (2) — deduplicate before analysis` |
| Overall null rate ≥ 20% | `High overall null rate (23.4%) — data may be incomplete` |

### Column-level findings

| Category | Example output |
|---|---|
| Null quality | `Moderate nulls (14.8%) — investigate cause before use` |
| High cardinality | `High cardinality (98.2%) — likely an identifier or natural key` |
| Constant column | `Constant column — single value throughout, no analytical value` |
| Low cardinality | `Very low cardinality (4 distinct values) — suitable for grouping or filtering` |
| Skewness | `Strongly right-skewed (3.12) — median is more representative than mean; consider log transform` |
| Negative values | `Contains 5 negative value(s) — verify this is expected for this field` |
| Dominant value | `Heavily dominated by 'Unknown' (91.3% of non-null values) — low informational value` |
| Long text | `Long average value length (74 chars) — may be free-text; consider text analysis` |

## Export

Both export buttons are **disabled** until a successful analysis has been run.

### Export JSON

Saves a single `.json` file containing:

```json
{
  "source_file": "sample_data.csv",
  "summary": { "rows": 27, "columns": 8, ... },
  "columns": { "customer_id": { "dtype": "int64", ... }, ... },
  "findings": {
    "table": [ "PRIMARY KEY CANDIDATE: 'customer_id' — ..." ],
    "columns": { "age": [ "Low null rate (3.7%) — minor gaps" ], ... }
  }
}
```

Default filename: `<source_stem>_profile.json`

### Export CSV

Saves a `.csv` file with one row per column plus a leading `__summary__` row.

Fields: `column, dtype, null_count, null_pct, unique_count, unique_pct, min, max, mean, median, mode, std, q1, q3, skewness, zeros, negatives, top_values, avg_length, findings`

- Numeric-only fields (min, max, mean, etc.) are blank for categorical columns
- `top_values` is serialised as `val1:n1; val2:n2; ...` for readability in Excel
- `findings` contains all column-level findings joined with ` | `

Default filename: `<source_stem>_profile.csv`

## Supported File Formats

| Format  | Extension  | Notes                                                   |
|---------|------------|---------------------------------------------------------|
| CSV     | `.csv`     | Loaded with `pandas.read_csv`                           |
| Parquet | `.parquet` | Loaded with `pandas.read_parquet` (requires `pyarrow`)  |

Files with any other extension will show an error message in the results panel.

## Column Statistics Reference

### All Columns

| Stat       | Description                                         |
|------------|-----------------------------------------------------|
| dtype      | pandas data type (e.g. `int64`, `float64`, `object`) |
| Null count | Number of missing values                            |
| Null %     | Percentage of rows that are null                    |
| Unique     | Number of distinct non-null values                  |
| Unique %   | Unique values as a percentage of total rows         |

### Numeric Columns (int, float — excluding bool)

| Stat      | Description                                              |
|-----------|----------------------------------------------------------|
| Min       | Smallest value                                           |
| Max       | Largest value                                            |
| Mean      | Arithmetic average                                       |
| Median    | Middle value (50th percentile)                           |
| Mode      | Most frequently occurring value                          |
| Std Dev   | Standard deviation (sample, ddof=1)                      |
| Q1        | 25th percentile                                          |
| Q3        | 75th percentile                                          |
| Skewness  | Asymmetry of distribution (highlighted if \|skew\| > 1)  |
| Zeros     | Count of values equal to 0                               |
| Negatives | Count of values less than 0                              |

### Categorical / String / Boolean Columns

| Stat         | Description                                        |
|--------------|----------------------------------------------------|
| Mode         | Most frequently occurring value                    |
| Avg length   | Mean character length of values (cast to string)   |
| Top 5 values | Most frequent values with their counts             |

## Customisation

### Adjusting Null Severity Thresholds

Edit the `_null_color` function in `profiler.py`:

```python
def _null_color(null_pct: float) -> str:
    if null_pct == 0:
        return "bold green"
    elif null_pct < 10:   # <-- change this threshold
        return "yellow"
    else:
        return "bold red"
```

The same thresholds are used by the findings engine in `interpreter.py` — update `_column_findings` there too if you want consistent messaging.

### Adjusting the Number of Top Values Shown

In `profile_column` inside `profiler.py`, change `.head(5)` to any number:

```python
top_values = non_null.value_counts().head(5).to_dict()  # change 5 to any n
```

## Sample Data

The included `sample_data.csv` contains 27 rows and 8 columns, intentionally designed to exercise all profiler and interpreter paths:

| Column      | Type    | Notes                                 |
|-------------|---------|---------------------------------------|
| customer_id | int64   | High cardinality — PK candidate       |
| name        | str     | 1 null, 2 duplicate values            |
| age         | float64 | 1 null                                |
| salary      | float64 | 2 nulls                               |
| department  | str     | Low cardinality (4 categories)        |
| score       | float64 | 2 nulls, slight negative skew         |
| join_date   | str     | Date strings (not parsed as datetime) |
| is_active   | bool    | 2-value boolean column                |

The last two rows are intentional duplicates of rows 2 and 10, so `Duplicate rows detected (2)` appears in the findings.
