# Python Data Profiler

A lightweight desktop GUI application for quickly profiling CSV and Parquet datasets. Select a file, click Analyse, and get statistical summaries, plain-English findings, and export options — all without writing any code.

---

## Features

- **Tkinter GUI** — file browser, results panel, export buttons; no command line needed
- Loads **CSV** and **Parquet** files into a pandas DataFrame
- **Azure SQL Server** connectivity — profile tables and views directly, or run a custom SQL query
- Computes per-column statistics including:
  - Null count and percentage
  - Unique count and percentage
  - Min, max, mean, median, mode, standard deviation
  - Q1 (25th percentile) and Q3 (75th percentile)
  - Skewness, zero count, negative count *(numeric columns)*
  - Top 5 most frequent values, average string length *(categorical/boolean columns)*
- Dataset-level summary: total rows, columns, nulls, duplicate rows, memory usage
- **Encoding detection** (CSV only) — automatically detects file encoding using charset-normalizer; confidence score and colour-coded status shown in results; low-confidence and fallback cases surfaced as findings
- **Findings & Recommendations** — plain-English interpretations generated by a rule-based engine:
  - Primary key, near-primary key, and composite key candidates
  - Per-column data quality warnings and usage notes
  - Encoding quality warnings
- **Export profile** results as JSON or CSV with a single click
- **Export clean data** as CSV or Parquet — applies automated cleaning and writes two audit logs alongside the output
- **Analysis timing** — elapsed time and per-stage breakdown shown at the top of every result
- Rich-formatted output (Unicode table borders, aligned columns, colour-coded severity) in the results panel

## Azure SQL Server Prerequisites

SQL Server connectivity requires two additional components installed separately from the main pip dependencies.

### Step 1 — Python package

```bash
pip install pyodbc
```

### Step 2 — Microsoft ODBC Driver 18 for SQL Server

This is a system-level driver that pyodbc uses to communicate with SQL Server. It is free and provided by Microsoft.

**Windows**

Download and run the `.msi` installer from:
[https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server)

To verify installation: open **ODBC Data Source Administrator** (`odbcad32.exe`) → **Drivers** tab → confirm "ODBC Driver 18 for SQL Server" is listed.

**macOS (Homebrew)**

```bash
brew tap microsoft/mssql-release https://github.com/Microsoft/homebrew-mssql-release
brew install msodbcsql18
```

**Linux (Ubuntu/Debian)**

```bash
curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -
curl https://packages.microsoft.com/config/ubuntu/$(lsb_release -rs)/prod.list \
  | sudo tee /etc/apt/sources.list.d/mssql-release.list
sudo apt-get update
sudo ACCEPT_EULA=Y apt-get install -y msodbcsql18
```

### Authentication notes

- **SQL auth** — enter your SQL Server username and password in the GUI connection panel
- **AAD Interactive** — when you click Analyse, a browser window will open for Azure AD sign-in (supports MFA). Complete the sign-in and the data fetch will proceed automatically.

### Firewall

Your Azure SQL Server's firewall must allow connections from your IP address. Configure this in the Azure portal under the SQL Server resource → **Networking**.

## Usage

Launch the application:

```bash
python main.py
```

A window will open. Select your data source using the **File** / **SQL Server** toggle at the top.

### Profiling a file

1. Select **File** (default)
2. Click **Browse** — choose a `.csv` or `.parquet` file
3. Click **Analyse**

### Profiling an Azure SQL Server table or view

1. Select **SQL Server**
2. Fill in **Server** (short name only, e.g. `myserver` — `.database.windows.net` is added automatically), **Database**, and **Auth** type
3. For SQL auth: enter **Username** and **Password**
4. Click **Connect** — the table/view list populates on success
5. Select a table or view from the dropdown, or type a custom SQL query in the **Custom SQL** box (custom SQL takes priority if both are provided)
6. Click **Analyse**

### After analysis

- **Export JSON** / **Export CSV** — save the profile results (enabled only after a successful analysis)
- **Export Clean CSV** / **Export Clean Parquet** — clean the data and save the output alongside two audit logs (enabled only after a successful analysis)
- Elapsed time and per-stage breakdown are shown at the top of the results panel
- The results panel is scrollable both vertically and horizontally, and re-renders at the current window width when Analyse is clicked

## Findings & Recommendations

The findings section appears above the statistics tables and is generated by a rule-based engine in `interpreter.py`. No external AI service is used — all analysis is local.

### Table-level findings

| Rule | Example output |
|---|---|
| Column with 100% unique non-null values | `PRIMARY KEY CANDIDATE: 'customer_id' — 100% unique, no nulls` |
| Column with ≥95% unique non-null values (no perfect PK found) | `NEAR-PRIMARY KEY: 'order_ref' — 97.2% unique, no nulls` |
| No single-column PK; pair/triple of columns is jointly unique | `COMPOSITE KEY CANDIDATE: ('col_a', 'col_b') — unique together` |
| No key of any kind detected | `No primary key or composite key candidate found` |
| Duplicate rows present | `Duplicate rows detected (2) — deduplicate before analysis` |
| Overall null rate ≥ 50% | `Critical overall null rate (62%) — dataset may be largely unusable` |
| Overall null rate ≥ 20% | `High overall null rate (23.4%) — data may be incomplete` |
| Encoding detection failed / forced fallback to Latin-1 | `Encoding detection failed — file was read as Latin-1 (fallback). Values may be garbled. Re-save the file as UTF-8 and reload.` |
| Encoding detected with low confidence (<85%) | `Encoding detected as 'windows-1252' with low confidence (71%). If values appear garbled, re-save the file as UTF-8 and reload.` |

### Column-level findings

| Category | Example output |
|---|---|
| Null quality | `Moderate nulls (14.8%) — investigate cause before use` |
| High cardinality | `High cardinality (98.2%) — likely an identifier or natural key` |
| Constant column | `Constant column — single value throughout, no analytical value` |
| Low cardinality | `Very low cardinality (4 distinct values) — suitable for grouping or filtering` |
| Skewness | `Strongly right-skewed (3.12) — median is more representative than mean; consider log transform` |
| Zero variance | `Zero variance — all non-null values are identical` |
| Negative values | `Contains 5 negative value(s) — verify this is expected for this field` |
| Zero values | `Contains 5 zero(s) — verify whether zero is a valid value or a substitute for null` |
| Dominant value | `Heavily dominated by 'Unknown' (91.3% of non-null values) — low informational value` |
| Long text | `Long average value length (74 chars) — may be free-text; consider text analysis` |

## Encoding Detection

For CSV files, the profiler automatically detects the file encoding using the `charset-normalizer` library and displays the result in the summary panel.

| Confidence | Display colour | Meaning |
|---|---|---|
| ≥ 85% | Default | Encoding detected reliably |
| < 85% | Yellow | Uncertain — values may be misread |
| 0% / fallback | Bold red | Detection failed; file read as Latin-1 |

Encoding issues are also surfaced in the **Findings** section with a recommendation to re-save the file as UTF-8.

Parquet files are binary and have no encoding to detect; they are always displayed as `binary (parquet)`.

## Export

All export buttons are **disabled** until a successful analysis has been run.

### Export JSON

Saves a single `.json` file containing:

```json
{
  "source_file": "sample_data.csv",
  "summary": { "rows": 27, "columns": 8, ... },
  "columns": { "customer_id": { "dtype": "int64", ... }, ... },
  "findings": {
    "table": [ "PRIMARY KEY CANDIDATE: 'customer_id' — ..." ],
    "columns": { "age": [ "Low null rate (3.7%) — minor gaps" ], ... }
  }
}
```

Default filename: `<source_stem>_profile.json`

### Export CSV

Saves a `.csv` file with one row per column plus a leading `__summary__` row.

Fields: `column, dtype, null_count, null_pct, unique_count, unique_pct, min, max, mean, median, mode, std, q1, q3, skewness, zeros, negatives, top_values, avg_length, findings`

- Numeric-only fields (min, max, mean, etc.) are blank for categorical columns
- `top_values` is serialised as `val1:n1; val2:n2; ...` for readability in Excel
- `findings` contains all column-level findings joined with ` | `

Default filename: `<source_stem>_profile.csv`

### Export Clean CSV / Export Clean Parquet

Applies automated cleaning to the dataset and writes three files to the chosen directory:

| File | Description |
|---|---|
| `<source_stem>_clean.csv` / `<source_stem>_clean.parquet` | The cleaned dataset |
| `<source_stem>_audit_log.csv` | Per-operation summary — operation type and affected row/column counts |
| `<source_stem>_audit_detail.csv` | Per-cell/row detail — original value and replacement value for every change |

## Supported File Formats

| Format  | Extension  | Notes                                                   |
|---------|------------|---------------------------------------------------------|
| CSV     | `.csv`     | Loaded with `pandas.read_csv`; encoding auto-detected   |
| Parquet | `.parquet` | Loaded with `pandas.read_parquet` (requires `pyarrow`)  |

Files with any other extension will show an error message in the results panel.

## Column Statistics Reference

### All Columns

| Stat       | Description                                         |
|------------|-----------------------------------------------------|
| dtype      | pandas data type (e.g. `int64`, `float64`, `object`) |
| Null count | Number of missing values                            |
| Null %     | Percentage of rows that are null                    |
| Unique     | Number of distinct non-null values                  |
| Unique %   | Unique values as a percentage of total rows         |

### Numeric Columns (int, float — excluding bool)

| Stat      | Description                                              |
|-----------|----------------------------------------------------------|
| Min       | Smallest value                                           |
| Max       | Largest value                                            |
| Mean      | Arithmetic average                                       |
| Median    | Middle value (50th percentile)                           |
| Mode      | Most frequently occurring value                          |
| Std Dev   | Standard deviation (sample, ddof=1)                      |
| Q1        | 25th percentile                                          |
| Q3        | 75th percentile                                          |
| Skewness  | Asymmetry of distribution (highlighted if \|skew\| > 1)  |
| Zeros     | Count of values equal to 0                               |
| Negatives | Count of values less than 0                              |

### Categorical / String / Boolean Columns

| Stat         | Description                                        |
|--------------|----------------------------------------------------|
| Mode         | Most frequently occurring value                    |
| Avg length   | Mean character length of values (cast to string)   |
| Top 5 values | Most frequent values with their counts             |

## Customisation

### Adjusting Null Severity Thresholds

Edit the `_null_color` function in `profiler.py`:

```python
def _null_color(null_pct: float) -> str:
    if null_pct == 0:
        return "bold green"
    elif null_pct < 10:   # <-- change this threshold
        return "yellow"
    else:
        return "bold red"
```

The same thresholds are used by the findings engine in `interpreter.py` — update `_column_findings` there too if you want consistent messaging.

### Adjusting the Number of Top Values Shown

In `profile_column` inside `profiler.py`, change `.head(5)` to any number:

```python
top_values = non_null.value_counts().head(5).to_dict()  # change 5 to any n
```
